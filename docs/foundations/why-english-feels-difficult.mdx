# Why English Feels Difficult

> **The real problem is usually not English itself. It is the way English has been learned.**

很多中文母语者学了十几年英语：背过单词、做过语法题，也通过了考试。

但当别人突然问：

> How's it going?

大脑仍然可能停顿一下。

你认识每一个单词，却不能自然地组织答案。这并不一定说明你“不擅长语言”。更可能的原因是：你学到的是许多彼此分离的规则，却没有建立一套能够连接这些规则的理解方式。

## Memorizing answers is not understanding a system

假设有人这样教你数学：

```text
2 + 3 = 5
7 × 8 = 56
12 − 4 = 8
```

他要求你背下越来越多的答案，却不解释加法、乘法和数量之间的关系。

你也许能够通过一次测试，但遇到一道从未见过的题目时，仍然不知道如何推导答案。

英语学习经常出现同样的问题：

- Present Perfect = `have + past participle`
- Passive Voice = `be + past participle`
- Third-person singular verbs add `-s`

这些规则可能都是正确的，但仅仅记住公式，并不会自动产生语言直觉。

因为规则告诉你 **what**，却没有告诉你 **why**。

## Grammar is the result, not the starting point

语言首先需要表达意义。语法是语言为了稳定地组织和传递意义而形成的模式。

所以，更有效的学习顺序不是：

```text
Rule → Memorize → Exercise
```

而是：

```text
Meaning → Mental Model → Pattern → Practice
```

先理解说话者想表达什么，再理解英语为什么选择这种结构，最后学习具体形式。

这也是 EB4CE 的基本方法：

> **Reason before rule.**

## A first mental model

不同语言会用不同方式组织同一个世界中的信息。

作为一个学习模型，可以先这样理解：在许多日常表达中，中文更容易把一部分信息留给上下文；英语则更倾向于把关键关系明确放进句子结构中。

```text
Chinese-oriented processing

Meaning
  ↓
Shared context fills in more information

English-oriented processing

Meaning
  ↓
The sentence makes more relationships explicit
```

这不是绝对的语言学定律。中文也可以非常明确，英语也高度依赖上下文。它只是一个有用的学习模型，用来帮助我们观察两种语言在很多常见场景中的不同倾向。

这个模型可以连接许多看似无关的问题：

- 为什么英语句子通常需要明确的主语？
- 为什么英语频繁使用代词？
- 为什么 `a`、`an` 和 `the` 会影响听者如何识别信息？
- 为什么被动语态可以改变句子的关注重点？

传统教材常把它们分成不同章节。建立心智模型后，你会开始看到它们之间的联系。

## Observation, model, and rule

EB4CE 会尽量区分三个层次。

### Observation

我们能够从真实英语中观察到什么？

例如：

> English finite clauses normally contain an explicit subject.

### Mental model

怎样理解这个现象最容易形成直觉？

例如：

> Think of the subject as the sentence's anchor.

“锚点”不是对主语的完整语言学定义，而是帮助学习者理解句子为何需要一个明确起点的模型。

### Rule

实际使用时，结构有什么限制？

例如：

> In standard English, we say **It is raining**, not simply **Is raining**.

把观察、模型和规则分开，可以避免两种问题：只背规则而不理解，或者把一个方便学习的类比误认为永远成立的事实。

## Understanding creates prediction

真正的语言能力，不只是记住见过的句子，而是能够处理没有见过的句子。

当你理解一个模式背后的原因时，就会开始预测：

- 一句话接下来可能需要什么；
- 为什么某种表达听起来完整；
- 为什么另一种表达虽然能翻译，却不自然；
- 在新场景中应该怎样组织信息。

这和理解一个软件系统很相似。你不需要背下每一次函数调用；理解接口和设计约束后，就能推断新的用法。

## One insight

这篇文章不要求你记住新的语法知识。

只需要保留一个新的问题：

> **不要只问：规则是什么？**  
> **还要问：英语为什么需要这个规则？**

当规则不再是孤立的事实，而是同一个系统中的结果，英语就会开始变得更容易理解，也更容易预测。

## Check your understanding

试着用自己的话回答：

1. 为什么背下大量正确的语法公式，仍然不一定能形成语言直觉？
2. “The subject is the sentence's anchor” 是观察、心智模型，还是语法规则？
3. 下面哪一种学习顺序更符合本章的方法？

```text
A. Rule → Memorize → Exercise
B. Meaning → Mental Model → Pattern → Practice
```

如果你能够解释原因，而不只是选择答案，你已经开始使用 EB4CE 的学习方式。
